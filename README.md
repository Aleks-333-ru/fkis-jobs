# Вакансии ФКиС

Сайт-агрегатор вакансий в сфере **физической культуры и спорта**.
Собирает вакансии из нескольких источников и показывает их с фильтрами:

- **hh.ru** — через открытый API HeadHunter (без ключа)
- **«Работа России»** — через открытый API trudvsem.ru (без ключа)
- **SuperJob** — по бесплатному ключу API (по желанию)

Маленький сервер на Node.js ходит в оба API, приводит вакансии к единому виду,
склеивает и убирает дубли. Браузер напрямую к этим API обращаться не может (CORS),
поэтому сервер обязателен для реальных данных.

## Как запустить

Нужен **Node.js версии 18 или новее** (проверить: `node -v`).
Скачать: https://nodejs.org

1. Положите папку проекта куда удобно.
2. В терминале перейдите в неё и выполните:

   ```bash
   node server.js
   ```

3. Откройте в браузере: **http://localhost:3000**

Никакой `npm install` не нужен — внешних зависимостей нет.

> Если открыть `public/index.html` напрямую (без сервера) — сайт покажет
> демо-данные и подсказку. Реальные вакансии появятся только при запущенном сервере.

## Как настроить под себя

Всё в `server.js`:

- `FKIS_TERMS` — список профессий для поиска в «Работа России».
- `HH_QUERY` — поисковый запрос для hh.ru (поддерживает `OR`, кавычки).
- `categorize()` — правила, как название вакансии попадает в категорию
  (Тренер / Учитель ФК / Инструктор / Фитнес / Методист).

Внешний вид и фильтры — в `public/index.html`.

## Дополнительный источник (SuperJob)

hh.ru и «Работа России» работают сразу, без ключей. SuperJob
подключается по бесплатному ключу API — пока ключ не задан, источник
просто отключён.

1. Получите ключ:
   - SuperJob — https://api.superjob.ru/register (нужен «Secret key»)
2. Впишите ключ в файл `.env` (скопируйте из `.env.example`):

   ```
   SUPERJOB_KEY=ваш_ключ
   ```

3. Запустите сервер с этим файлом:

   ```bash
   node --env-file=.env server.js
   ```

4. На сайте включите галочку **SuperJob** в блоке «Источник».

> Файл `.env` с реальными ключами не коммитится в git (он в `.gitignore`).
> На хостинге ключи задаются в разделе Environment Variables, а не в `.env`.

## Где разместить в интернете

Подойдёт любой хостинг с поддержкой Node.js: Render, Railway, Beget, VPS и т.п.
Сервер слушает порт из переменной `PORT`, команда запуска — `npm start`
(то есть `node server.js`).

### Деплой на Render (бесплатно)

1. Загрузите проект в репозиторий на GitHub.
2. На https://render.com → **New → Web Service** → подключите репозиторий.
3. Настройки:
   - **Runtime:** Node
   - **Build Command:** *(пусто, зависимостей нет)*
   - **Start Command:** `npm start`
4. (Опционально) в **Environment → Add Environment Variable** добавьте
   `SUPERJOB_KEY`.
5. **Create Web Service** → через пару минут сайт будет доступен по адресу
   вида `https://имя-сервиса.onrender.com`.
